# Status Update Output Formats

Templates for the four audience versions. Generated by `/status-update`.
Only generate the versions the user selected.

---

## Client-Facing

Tone: polished, confident, professionally warm. Use `tone-of-voice.md` if available.
Frames progress positively without lying. Translates internal jargon into client
language. Blockers are reframed as "areas where we need your input" where appropriate.

```markdown
### {Project Name} — Status Update — {Date}

**Progress this week:**
- {update 1}
- {update 2}

**What we need from you:**
- {action or decision needed — pull from Open Decisions where owner is the client}

**Coming up next:**
- {next milestone or deliverable — pull from Timeline section of state}
```

---

## Leadership

Tone: direct, concise, honest about risks. No sugar-coating. Written for someone
scanning 15 updates in 10 minutes.

```markdown
### {Project Name} — Leadership Update — {Date}

**Status:** {On Track / At Risk / Off Track}
**Trend:** {↑ Improving / → Steady / ↓ Declining} {context — e.g., "second week at risk", "moved from At Risk to On Track"}

**This week:**
- {update 1}
- {update 2}

**Timeline Status:**
{Only if Timeline section in state has data. Otherwise omit.}
- Current phase: {phase} — {on schedule / behind by Xd / ahead}
- Next milestone: {milestone} — {date} — {on track / at risk}
- Slip scenario materialising? {Yes — describe / No}

**Budget & Pace:**
{Only if Budget & Pace section in state has data. Otherwise omit.}
- Hours: {actual} / {budgeted} ({%})
- Burn rate: {hrs/week} ({trend})
- Margin: {healthy / thinning / at risk}
- Projection: {e.g., "At current pace, budget exhausted by [date] with [deliverables] remaining"}

**Flags:**
- {risks from state Risks section, plus any new ones from this week's paste}

**Client Engagement Health:** {GREEN / AMBER / RED}
{Only if Engagement Health section has signals. Otherwise omit.}
- {signal 1}
- {signal 2}
- **Recommended action:** {e.g., "MD calls client CMO for informal catch-up this week"}

**Open Decisions:**
{Pull from state Open Decisions section. Show pending duration prominently —
accountability comes from visibility.}
- {decision} — raised {date} ({Nd pending}) — owner: {who}

**Decisions Resolved This Week:**
{Only if any open decisions were resolved this week.}
- {decision} — resolved: {outcome}
```

## Creative Team

Tone: practical, no fluff. What changed, what's coming, what they need to do
differently. Respects their time and intelligence.

```markdown
### {Project Name} — Team Update — {Date}

**What changed:**
- {decision or direction shift that affects creative work}

**What's next for you:**
- {specific deliverable or task with deadline}

**Watch out for:**
- {anything that might change again, client feedback patterns, capacity risks}
```

---

## Slack Format

Compact, single-message. No tables. Bold and line breaks for scannability.

```markdown
*{Project Name} — Week of {Date}*

**Status: {On Track / At Risk / Off Track}** {↑/→/↓}

*This week:* {1-2 sentence summary}
*Blockers:* {1 sentence or "None"}
*Next up:* {next milestone and date}
*Needs:* {any decisions or client input needed}

{If Engagement Health is AMBER or RED, add one line:}
*Client health: {AMBER/RED} — {one-line signal}*
```
